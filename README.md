# Subflow-Utility
Subflow to wait for all the CTASKS to get completed

✓ Developed a reusable subflow to wait until all catalog tasks are either closed complete or closed
incomplete.
✓ Enabled integration of this subflow with any flow requiring catalog task completion tracking.
✓ Implemented logic to monitor active tasks, ensuring execution pauses until task completion
before returning a success value.
✓ Designed the subflow to return a false value when no active tasks are found, improving flow
accuracy and reliability.
✓ Subflow inputs: ritm – Reference to requested item (sc_req_item)
✓ Subflow outputs: Output – Boolean
